## Overview

The NoteMV editor is a powerful, Notion-inspired rich text editor built on Quill. It provides a distraction-free writing environment with advanced formatting, real-time collaboration, and version control.

## Opening the Editor

The editor opens in several ways:

<Tabs>
  <Tab title="Quick Notes">
    From the **Dashboard**:
    - Click the **"+" card** for new note
    - Click any **existing note card** to edit
  </Tab>
  
  <Tab title="Project Documents">
    From a **Project**:
    - Click **"📝+ Nuevo Doc"** button
    - Click any **existing document** in the file browser
  </Tab>
</Tabs>

<Note>
  The editor always opens in full-screen mode with the sidebar hidden, giving you maximum writing space.
</Note>

## Editor Interface

### Top Navigation Bar

The header provides essential controls and status:

```

                              [Users] Guardado  ⏳  [Auto] 🖨️ Guardar
                                      ↓ status
```

**Left Side:**
- **⬅ Volver** - Return to previous view (saves automatically)

**Right Side:**
- **Active Users** - Avatars of collaborators currently viewing

- **Last Save Time** - Timestamp of most recent save
- **⏳ Time Machine** - Open version history
- **Auto-Save Toggle** - Enable/disable automatic saving
- **🖨️ PDF Export** - Generate and download PDF
- **Guardar Button** - Manual save trigger

### Image Management

<Accordion title="Adding Images">
  **Method 1: Toolbar Button**
  1. Click the image icon in toolbar
  2. Enter image URL when prompted
  3. Image inserts at cursor position
  
  **Method 2: Cover Image Gallery**
  1. Hover at top of document
  2. Click **"🖼️ Agregar Portada"**
  3. Search for images by keyword
  4. Click to insert (uses Flickr random images)
  
  **Image Styling:**
  - Automatically centered
  - Max width to fit page
  - 8px rounded corners
  - Subtle shadow effect
</Accordion>

<Warning>
  Images must be hosted externally. NoteMV uses URLs, not file uploads. Consider using Imgur, Cloudinary, or your own CDN.
</Warning>

## Auto-Save System

### How It Works

<Steps>
  <Step title="Type Normally">
    Make changes to title, content, or tag
  </Step>
  
  <Step title="2-Second Delay">
    Auto-save waits 2 seconds after your last keystroke
  </Step>
  
  <Step title="Saves to Firestore">
    Changes sync to database in the background
  </Step>
  
  <Step title="Status Updates">
    Header shows "Guardando..." then "Guardado"
  </Step>
</Steps>

**Auto-Save Toggle:**

The switch in the header controls auto-save:
- **Green** - Auto-save enabled (recommended)
- **Gray** - Auto-save disabled (manual only)

<Tip>
  Leave auto-save enabled to prevent data loss. Manual save is still available anytime.
</Tip>

## Version History (Time Machine)

### Automatic Snapshots

NoteMV creates version snapshots automatically:

- **Every 20 minutes** while editing
- **On exit** when leaving the editor
- **Manual saves** can also create versions

### Accessing History

Click the **⏳** icon in the header to open Time Machine:

<Steps>
  <Step title="View Version List">
    Modal shows all saved versions with timestamps
  </Step>
  
  <Step title="Preview Changes">
    Click a version to see the diff or full content
  </Step>
  
  <Step title="Restore Version">
    Click "Restaurar" to revert to that version
  </Step>
  
  <Step title="Confirmation">
    Document restores and saves immediately
  </Step>
</Steps>

<Accordion title="Version Metadata">
  Each version stores:
  - **Timestamp** - When snapshot was created
  - **User** - Who made the changes
  - **Title** - Document title at that time
  - **Content** - Full HTML content
  - **Label** - Auto reason ("Auto - 20min", "Auto - Al salir")
</Accordion>

<Warning>
  Restoring a version **overwrites** the current document. This creates a new auto-save immediately.
</Warning>

## Export and Printing

### PDF Generation

Click **🖨️ PDF** to export:

<Tabs>
  <Tab title="Print Dialog">
    Opens browser's native print dialog:
    - Choose printer or "Save as PDF"
    - Adjust margins and layout
    - Select pages or page range
  </Tab>
  
  <Tab title="Styling">
    PDF export uses print-specific CSS:
    - White background (not dark theme)
    - Black text for readability
    - Document title as page header
    - Clean, professional formatting
    - Images scale appropriately
  </Tab>
</Tabs>

**PDF Features:**
- Title appears as underlined header
- Toolbar hidden automatically
- Emoji icon removed from print
- Metadata (date/tag) excluded
- Clean, distraction-free output

<Tip>
  Use your browser's print preview to verify formatting before saving as PDF.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Changes Not Saving">
    **Check:**
    - Auto-save toggle is enabled (green)
    - You have Editor or Owner permissions
    - Internet connection is stable
    - No error in browser console
    
    **Solution:**
    - Click manual "Guardar" button
    - Refresh page (auto-save continues)
    - Check project permissions with owner
  </Accordion>
  
  <Accordion title="Collaborator Not Appearing">
    **Possible causes:**
    - They haven't opened the document yet
    - Presence subscription hasn't initialized
    - Firebase connection issue
    
    **Solution:**
    - Wait 5-10 seconds for sync
    - Have them refresh the page
    - Check project access permissions
  </Accordion>
  
  <Accordion title="Version History Empty">
    **Reasons:**
    - Document created recently (no 20min snapshots yet)
    - First time editing this document
    - History feature disabled for quick notes
    
    **Workaround:**
    - Keep editing; history builds over time
    - Manual save creates checkpoint
    - Exit and re-enter to trigger save
  </Accordion>
</AccordionGroup>
