## Overview

Projects in NoteMV are collaborative workspaces where teams can organize documents, manage tasks, and communicate in real-time. Each project acts as a container for related content with full permission management.

## Creating Projects

<Steps>
  <Step title="Navigate to Projects">
    Click **"Mis Proyectos"** in the sidebar
  </Step>
  
  <Step title="Click '+ PROYECTO'">
    Green button in the top-right corner
  </Step>
  
  <Step title="Name Your Project">
    Enter a descriptive project name
  </Step>
  
  <Step title="Start Organizing">
    Your new project opens immediately, ready for content
  </Step>
</Steps>

<Note>
  You are automatically assigned as the **Owner** of projects you create, giving you full control over settings and permissions.
</Note>

**Creating Documents:**

<Steps>
  <Step title="Click '📝+ Nuevo Doc'">

### Context Menu Actions

Right-click (or click •••) on any item for options:

| Action | Description | Permission Required |
|--------|-------------|---------------------|
| ✏ Editar Información | Edit project metadata | Editor or Owner |
| 📦 Mover a... | Move document to different folder/project | Editor or Owner |
| 🗑️ Papelera | Move to trash | Editor or Owner |

<Warning>
  Only project **Owners** can delete entire projects. Editors can only manage documents and folders.
</Warning>

## Team Collaboration

### Permission Roles

NoteMV uses a three-tier permission system:

<Tabs>
  <Tab title="Owner (Dueño)">
    **Full Control** - Only one per project
    
    - Edit project information
    - Invite and remove collaborators
    - Change user roles
    - Delete the project
    - All editor capabilities
  </Tab>
  
  <Tab title="Editor">
    **Read + Write** - Can modify content
    
    - Create, edit, and delete documents
    - Create and organize folders
    - Move files between folders
    - Create and manage tasks
    - Participate in chat
  </Tab>
  
  <Tab title="Visor (Viewer)">
    **Read Only** - Cannot make changes
    
    - View all documents
    - Browse folder structure
    - See tasks and progress
    - Read chat messages
    - Cannot edit or create content
  </Tab>
</Tabs>

### Inviting Team Members

<Steps>
  <Step title="Click 'Invitar' Button">
    Available in project header (Owner only)
  </Step>
  
  <Step title="Enter Email Address">
    Type the collaborator's email
  </Step>
  
  <Step title="Select Role">
    Choose between Visor (viewer) or Editor
  </Step>
  
  <Step title="Copy Invitation Link">
    System generates a shareable link with message
  </Step>
</Steps>

<Tip>
  The invitation link includes the platform URL and project name. Collaborators must have a NoteMV account to access shared projects.
</Tip>

### Managing Team Members

Click **"👥 Equipo"** to view and manage the team:

**Team Management Panel:**

```
👑 owner@email.com         [Propietario]
───────────────────────────────────────
✏️ editor@email.com        [Editor ▼] 🗑️
   Agregado: 12/01/2024

👁️ viewer@email.com        [Visor ▼] 🗑️
   Agregado: 15/01/2024
```

**Available Actions:**
- **Change Role** - Dropdown to switch between Editor/Visor
- **Remove User** - Click trash icon to revoke access
- **View Join Date** - See when each member was added

<Note>
  Real-time subscriptions ensure permission changes take effect immediately. If a user loses access, they're automatically kicked from the project.
</Note>

## Moving Content

### Moving Documents Between Projects

<Steps>
  <Step title="Open Context Menu">
    Click ••• on a document
  </Step>
  
  <Step title="Select 'Mover a...'">
    Opens the move dialog
  </Step>
  
  <Step title="Choose Destination">
    Select target project and folder (or root)
  </Step>
  
  <Step title="Confirm Move">
    Document relocates with full history preserved
  </Step>
</Steps>

<Warning>
  You can only move documents to projects where you have **Editor** or **Owner** permissions.
</Warning>

  <Card title="Document Changes">

### Project Chat

A floating chat window is available throughout the project:

<Accordion title="Terminal-Style Chat Interface">
  **Features:**
  - Terminal aesthetic with green borders
  - Fixed position in bottom-right corner
  - Collapsible to save screen space
  - Red notification badge when minimized
  - Scroll-to-bottom on new messages
  - User identification with color-coded prompts
  
  **Usage:**
  ```
  root@local:~$ Message from owner
  editor@local:~$ Response from editor
  ```
</Accordion>

<Tip>
  Chat history is persistent and loaded from Firestore in real-time, so new members can see previous conversations.
</Tip>

## Project Workflows

<AccordionGroup>
  <Accordion title="Documentation-Heavy Projects">
    1. Create folder structure (Frontend, Backend, Docs, etc.)
    2. Add documents to appropriate folders
    3. Use external Documentos tab for references
    4. Keep chat open for quick clarifications
  </Accordion>
  
  <Accordion title="Task-Focused Projects">
    1. Start with Tareas tab for sprint planning
    2. Create tasks assigned to team members
    3. Use documents for specs and requirements
    4. Track progress with metrics dashboard
  </Accordion>
  
  <Accordion title="Hybrid Approach">
    1. Organize documents by feature/sprint
    2. Create corresponding tasks in Kanban
    3. Link documents in task descriptions
    4. Use chat for standup updates
  </Accordion>
</AccordionGroup>

## Best Practices

<CardGroup cols={2}>
  <Card title="Naming Conventions" icon="tag">
    - Use clear, descriptive project names
    - Prefix folders by category or phase
    - Keep document titles concise
    - Tag documents consistently
  </Card>
  
  <Card title="Permission Management" icon="shield">
    - Start with Visor for new members
    - Promote to Editor when trust is established
    - Review team list regularly
    - Remove inactive collaborators
  </Card>
  
  <Card title="Organization" icon="folder-tree">
    - Don't nest folders more than 3 levels deep
    - Archive completed work to separate folders
    - Use consistent folder structure across projects
    - Delete empty or unused folders
  </Card>
  
  <Card title="Communication" icon="comment">
    - Use chat for quick questions
    - Document important decisions in notes
    - Reference documents in chat messages
    - Keep chat professional and on-topic
  </Card>
</CardGroup>
