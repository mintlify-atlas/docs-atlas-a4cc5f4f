## Overview

`ProjectChat` is a floating chat widget that provides real-time messaging for project teams. It features a terminal-inspired design and appears as a floating button that expands into a chat window.

## Usage

```jsx
import ProjectChat from './componentes/views/ProjectChat';

<ProjectChat
  proyectoId={project.id}
  user={currentUser}
/>
```

## Props

<ParamField path="proyectoId" type="string" required>
  The project ID to subscribe to for chat messages.
</ParamField>

<ParamField path="user" type="object" required>
  Current user object for sending messages.
  
  **Required properties**:
  - `uid`: User ID for message identification
  - `displayName`: Username displayed in chat
  - `email`: User email
</ParamField>

## State Management

### Core States

<ResponseField name="abierto" type="boolean" default="false">
  Controls whether the chat window is expanded or minimized.
</ResponseField>

<ResponseField name="mensajes" type="array" default="[]">
  Array of message objects from Firestore subscription.
</ResponseField>

<ResponseField name="input" type="string" default="''">
  Current message input value.
</ResponseField>

<ResponseField name="tieneNotificacion" type="boolean" default="false">
  Shows red notification badge when new messages arrive while closed.
</ResponseField>

## Real-time Messaging

### Subscribing to Chat

```jsx
useEffect(() => {
  if (!proyectoId) return;
  
  const unsubscribe = suscribirChat(proyectoId, (msgs) => {
    setMensajes(prevMsgs => {
      // Show notification if new messages arrive while chat is closed
      if (msgs.length > prevMsgs.length && 
          !abiertoRef.current && 
          !isFirstLoad.current) {
        setTieneNotificacion(true);
      }
      return msgs;
    });
    
    if (isFirstLoad.current) isFirstLoad.current = false;
  });
  
  return () => unsubscribe();
}, [proyectoId]);
```

### Sending Messages

```jsx
const handleSubmit = async (e) => {
  e.preventDefault();
  if (!input.trim()) return;
  
  await enviarMensaje(proyectoId, user, input);
  setInput("");
};
```

## UI States

### Closed State (Floating Button)

When `abierto === false`, displays a minimized floating button:

```jsx
<button 
  onClick={() => setAbierto(true)}
  className="fixed bottom-6 right-6 bg-black border border-green-500 
             w-14 h-14 rounded-full flex items-center justify-center 
             shadow-[0_0_15px_rgba(34,197,94,0.4)] hover:scale-110"
>
  {tieneNotificacion && (
    <span className="absolute top-0 right-0 flex h-4 w-4">
      <span className="animate-ping absolute inline-flex h-full w-full 
                       rounded-full bg-red-400 opacity-75"></span>
      <span className="relative inline-flex rounded-full h-4 w-4 
                       bg-red-500 border-2 border-black"></span>
    </span>
  )}
  {'>_'}
</button>
```

### Open State (Chat Window)

When `abierto === true`, displays full chat interface:

- **Title bar**: Shows project ID and close button
- **Message area**: Scrollable message history
- **Input bar**: Terminal-style message input

## Message Display

Messages are rendered with terminal-style formatting:

```jsx
{mensajes.map((msg) => {
  const esMio = msg.uid === user.uid;
  return (
    <div key={msg.id} className="flex flex-col animate-fadeIn">
      <span className={`text-[10px] ${
        esMio ? 'text-blue-400' : 'text-purple-400'
      }`}>
        {esMio 
          ? 'root' 
          : msg.usuario.toLowerCase().replace(' ', '_')
        }@local:~$
      </span>
      <span className="text-green-300 break-words pl-2">
        {msg.texto}
      </span>
    </div>
  );
})}
```

## Auto-scroll Behavior

Automatically scrolls to latest message:

```jsx
const bottomRef = useRef(null);

// Scroll on open
useEffect(() => {
  if (abierto) {
    setTimeout(() => {
      bottomRef.current?.scrollIntoView({ behavior: "smooth" });
    }, 100);
  }
}, [abierto]);

// Scroll on new messages
useEffect(() => {
  if (abierto) {
    bottomRef.current?.scrollIntoView({ behavior: "smooth" });
  }
}, [mensajes, abierto]);
```

## Notification Logic

The notification system uses refs to track state accurately:

```jsx
const abiertoRef = useRef(abierto);
const isFirstLoad = useRef(true);

useEffect(() => {
  abiertoRef.current = abierto;
  if (abierto) {
    setTieneNotificacion(false); // Clear notification when opened
  }
}, [abierto]);
```

## Terminal Styling

The component uses terminal-inspired design:

- Monospace font (`font-mono`)
- Green text on black background
- Command prompt-style input (`>` prefix)
- SSH connection header message
- Matrix-style color scheme

### Header

```jsx
<div className="bg-green-900/20 border-b border-green-500/50 p-2">
  <span className="text-green-400 text-xs tracking-widest">
    root@chat:~/{proyectoId.slice(0,6)}
  </span>
  <button onClick={() => setAbierto(false)}>x</button>
</div>
```

### Input Bar

```jsx
<form className="p-2 border-t border-green-500/30 bg-black flex items-center gap-2">
  <span className="text-green-500 animate-pulse">{'>'}</span>
  <input 
    type="text" 
    value={input}
    onChange={(e) => setInput(e.target.value)}
    className="flex-1 bg-transparent text-white outline-none"
    placeholder="Escribir comando..."
  />
</form>
```

## Positioning

Fixed positioning ensures chat is always accessible:

```jsx
className="fixed bottom-6 right-6 w-96 h-[500px] z-50"
```

<Note>
The chat uses `useRef` for the `abierto` state to prevent stale closures in the subscription callback, ensuring notifications work correctly.
</Note>

<Note>
The `isFirstLoad` ref prevents showing a notification when the component first mounts and loads existing messages.
</Note>

<Warning>
Ensure the parent container has `position: relative` to properly contain the fixed-position chat widget within the project context.
</Warning>
