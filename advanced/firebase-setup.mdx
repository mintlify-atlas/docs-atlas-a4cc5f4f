# Firebase Configuration

NoteMV uses Firebase as its backend infrastructure for authentication, real-time database, and cloud storage. This guide walks you through the complete setup process.

## Prerequisites

Before starting, ensure you have:
- A Google account
- Node.js 16+ installed
- Access to [Firebase Console](https://console.firebase.google.com)

## Initial Firebase Setup

### 1. Create a Firebase Project

1. Navigate to the [Firebase Console](https://console.firebase.google.com)
2. Click **Add Project**
3. Enter your project name (e.g., `notemv-production`)
4. Enable Google Analytics (optional but recommended)
5. Wait for project creation to complete

### 2. Register Your Web App

1. In your Firebase project, click the **Web** icon (`</>`)
2. Register your app with a nickname (e.g., "NoteMV Web")
3. Copy the Firebase configuration object

<Warning>
  Never commit your Firebase configuration with real credentials to public repositories. Use environment variables for production deployments.
</Warning>

## Configuration Structure

NoteMV's Firebase configuration is located in `src/firebase.js`. Here's the complete structure:

```javascript src/firebase.js
import { initializeApp } from "firebase/app";
import { getAuth, GoogleAuthProvider, signInWithPopup, signOut } from "firebase/auth";
import { getFirestore } from "firebase/firestore";

// Your web app's Firebase configuration
const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  authDomain: "your-project.firebaseapp.com",
  projectId: "your-project-id",
  storageBucket: "your-project.firebasestorage.app",
  messagingSenderId: "YOUR_SENDER_ID",
  appId: "YOUR_APP_ID"
};

// Initialize Firebase
const app = initializeApp(firebaseConfig);
const auth = getAuth(app);
const googleProvider = new GoogleAuthProvider();
const db = getFirestore(app);

export { auth, googleProvider, signInWithPopup, signOut, db };
```

### Configuration Parameters

| Parameter | Description | Example |
|-----------|-------------|----------|
| `apiKey` | Public API key for Firebase services | `AIzaSyB3ya...` |
| `authDomain` | Domain for authentication redirects | `notemv.firebaseapp.com` |
| `projectId` | Unique project identifier | `notemv-72ca1` |
| `storageBucket` | Cloud Storage bucket URL | `notemv.firebasestorage.app` |
| `messagingSenderId` | FCM sender ID for push notifications | `389278295908` |
| `appId` | Unique app identifier | `1:389278...` |

## Firestore Security Rules

Implement these security rules in the Firebase Console under **Firestore Database > Rules**:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Helper function to check authentication
    function isAuthenticated() {
      return request.auth != null;
    }
    
    // Helper function to check ownership
    function isOwner(userId) {
      return isAuthenticated() && request.auth.uid == userId;
    }
    
    // Helper function to check project access
    function hasProjectAccess(projectData) {
      return isAuthenticated() && 
             (request.auth.uid == projectData.userId || 
              request.auth.token.email in projectData.accesos);
    }
    
    // Notes collection
    match /notas/{notaId} {
      allow read: if isAuthenticated() && 
                     (resource.data.userId == request.auth.uid ||
                      resource.data.proyectoId != null);
      allow create: if isAuthenticated() && 
                       request.resource.data.userId == request.auth.uid;
      allow update, delete: if isOwner(resource.data.userId);
      
      // Presence subcollection (real-time collaboration)
      match /presencia/{userId} {
        allow read: if isAuthenticated();
        allow write: if isAuthenticated() && userId == request.auth.uid;
      }
    }
    
    // Projects collection
    match /proyectos/{proyectoId} {
      allow read: if hasProjectAccess(resource.data);
      allow create: if isAuthenticated() && 
                       request.resource.data.userId == request.auth.uid;
      allow update: if hasProjectAccess(resource.data);
      allow delete: if isOwner(resource.data.userId);
      
      // Project chat subcollection
      match /mensajes/{mensajeId} {
        allow read: if isAuthenticated();
        allow create: if isAuthenticated();
        allow delete: if isOwner(resource.data.uid);
      }
    }
    
    // Tasks collection
    match /tareas/{tareaId} {
      allow read, write: if isAuthenticated();
    }
    
    // Folders collection
    match /carpetas/{carpetaId} {
      allow read, write: if isAuthenticated();
    }
  }
}
```

<Info>
  These security rules provide basic protection. For production environments, implement more granular access controls based on project ownership and collaboration roles.
</Info>

## Authentication Setup

### Enable Google Sign-In

1. In Firebase Console, navigate to **Authentication > Sign-in method**
2. Click **Google** provider
3. Toggle **Enable**
4. Add your support email
5. Save changes

### Authentication Implementation

NoteMV uses Google OAuth for authentication. Here's how it's implemented:

```javascript src/App.jsx
import { auth, googleProvider, signInWithPopup, signOut } from './firebase';

function App() {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(false);

  // Login with Google
  const handleLogin = async () => {
    setLoading(true);
    try {
      const result = await signInWithPopup(auth, googleProvider);
      setUser(result.user);
    } catch (error) {
      console.error("Error al iniciar sesión:", error);
    } finally {
      setLoading(false);
    }
  };

  // Logout
  const handleLogout = async () => {
    await signOut(auth);
    setUser(null);
  };

  return (
    <>
      {user ? (
        <DashboardLayout user={user} onLogout={handleLogout} />
      ) : (
        <WelcomeScene onEnter={handleLogin} />
      )}
    </>
  );
}
```

## Real-Time Features

NoteMV leverages Firestore's real-time capabilities for collaboration features.

### Real-Time Note Synchronization

```javascript src/servicios/notas.js
import { onSnapshot, doc } from "firebase/firestore";
import { db } from "../firebase";

export const suscribirNota = (notaId, callback) => {
  if (!notaId) return;
  
  const unsubscribe = onSnapshot(doc(db, "notas", notaId), (doc) => {
    if (doc.exists()) {
      callback({ ...doc.data(), id: doc.id });
    }
  });

  return unsubscribe;
};
```

### Presence System

Track active users viewing a note:

```javascript
import { setDoc, deleteDoc, serverTimestamp } from "firebase/firestore";

export const registrarPresencia = async (notaId, user) => {
  if (!notaId || !user) return;
  try {
    const presenciaRef = doc(db, "notas", notaId, "presencia", user.uid);
    await setDoc(presenciaRef, {
      uid: user.uid,
      displayName: user.displayName || "Anónimo",
      photoURL: user.photoURL || "",
      email: user.email,
      color: '#' + Math.floor(Math.random()*16777215).toString(16),
      conectado: serverTimestamp()
    });
  } catch (e) { 
    console.error("Error al registrar presencia:", e); 
  }
};
```

<Note>
  Remember to clean up presence when users navigate away by calling `retirarPresencia()`.
</Note>

## Environment Variables (Production)

For production deployments, use environment variables:

```bash .env
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_auth_domain
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_storage_bucket
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id
```

Update your firebase.js:

```javascript
const firebaseConfig = {
  apiKey: import.meta.env.VITE_FIREBASE_API_KEY,
  authDomain: import.meta.env.VITE_FIREBASE_AUTH_DOMAIN,
  projectId: import.meta.env.VITE_FIREBASE_PROJECT_ID,
  storageBucket: import.meta.env.VITE_FIREBASE_STORAGE_BUCKET,
  messagingSenderId: import.meta.env.VITE_FIREBASE_MESSAGING_SENDER_ID,
  appId: import.meta.env.VITE_FIREBASE_APP_ID
};
```

## Testing Your Configuration

Verify your Firebase setup:

1. Start the development server: `npm run dev`
2. Open the browser console
3. Attempt to sign in with Google
4. Check for any Firebase errors in the console
5. Verify user authentication state in Firebase Console > Authentication

<Warning>
  If you see CORS errors, ensure your domain is authorized in Firebase Console > Authentication > Settings > Authorized domains.
</Warning>
