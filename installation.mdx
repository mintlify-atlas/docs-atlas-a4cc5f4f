## Configure Firebase

NoteMV uses Firebase for authentication, database, and hosting. Follow these steps to set up your Firebase project.

<Steps>
  <Step title="Create a Firebase Project">
    1. Go to the [Firebase Console](https://console.firebase.google.com/)
    2. Click **Add project**
    3. Enter a project name (e.g., "notemv-dev")
    4. Follow the setup wizard
  </Step>

  <Step title="Enable Authentication">
    1. In the Firebase console, navigate to **Authentication**
    2. Click **Get started**
    3. Enable **Google** as a sign-in provider
    4. Add your development domain to authorized domains
  </Step>

  <Step title="Create a Firestore Database">
    1. Navigate to **Firestore Database**
    2. Click **Create database**
    3. Choose **Start in test mode** for development
    4. Select a region close to your users

    <Warning>
      Remember to configure security rules before deploying to production!
    </Warning>
  </Step>

  <Step title="Get Firebase Configuration">
    1. Go to **Project Settings** (gear icon)
    2. Scroll to **Your apps** section
    3. Click the web icon (`</>`) to add a web app
    4. Copy your Firebase configuration object
  </Step>
</Steps>

## Update Firebase Configuration

Replace the Firebase configuration in `src/firebase.js` with your project credentials:

```javascript src/firebase.js
import { initializeApp } from "firebase/app";
import { getAuth, GoogleAuthProvider, signInWithPopup, signOut } from "firebase/auth";
import { getFirestore } from "firebase/firestore";

// Replace with your Firebase configuration
const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  authDomain: "YOUR_PROJECT_ID.firebaseapp.com",
  projectId: "YOUR_PROJECT_ID",
  storageBucket: "YOUR_PROJECT_ID.firebasestorage.app",
  messagingSenderId: "YOUR_MESSAGING_SENDER_ID",
  appId: "YOUR_APP_ID"
};

// Initialize Firebase
const app = initializeApp(firebaseConfig);
const auth = getAuth(app);
const googleProvider = new GoogleAuthProvider();
const db = getFirestore(app);

export { auth, googleProvider, signInWithPopup, signOut, db };
```

<Warning>
  **Security Best Practice**: Never commit Firebase credentials to version control. Use environment variables for production.
</Warning>

## Run the Development Server

Start the Vite development server:

```bash
npm run dev
```

You should see output similar to:

```bash
VITE v7.2.4  ready in 123 ms

➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
```

Open [http://localhost:5173](http://localhost:5173) in your browser.

<Note>
  The dev server includes hot module replacement (HMR) - changes to your code will update instantly without refreshing the page.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Firebase authentication fails">
    **Solution:**
    - Verify your Firebase configuration is correct
    - Check that Google authentication is enabled in Firebase Console
    - Add `localhost:5173` to authorized domains in Firebase Authentication settings
  </Accordion>

  <Accordion title="Module not found errors">
    **Solution:**
    ```bash
    # Clear node_modules and reinstall
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>

  <Accordion title="Firestore permission denied">
    **Solution:**
    - Ensure Firestore rules allow read/write for authenticated users
    - For development, use test mode:
    ```javascript
    rules_version = '2';
    service cloud.firestore {
      match /databases/{database}/documents {
        match /{document=**} {
          allow read, write: if request.auth != null;
        }
      }
    }
    ```
  </Accordion>

  <Accordion title="3D scene not rendering">
    **Solution:**
    - Check browser console for WebGL errors
    - Ensure your browser supports WebGL
    - Try disabling hardware acceleration in browser settings and re-enabling it
  </Accordion>
</AccordionGroup>
